# mango-money 迁移到 mango-fleetflow-ai 说明

目标：把当前 `mango-money` 财务系统拆解并迁移到 `mango-fleetflow-ai` React 项目中，最终在 React 项目的「财务管理」页面内承载完整财务功能。

迁移后的财务管理 Tab 建议包含：

```text
收支流水
营收日报
应收账款
押金流水
工资提成
中台
定金单
尾款单
对账单
历史图片
推送
账号管理
表格管理
```

## 一、迁移总原则

### 1. 数据库不迁移

两个项目当前都使用同一个数据库：

```text
cardata_money
```

因此业务数据、表格配置、字段配置、公式配置不需要搬库。React 项目继续连接同一个 `cardata_money` 即可。

### 2. 后端逻辑要拆分迁移

`mango-money` 当前后端集中在：

```text
server.js
```

迁移时不要把整个 `server.js` 直接复制过去运行，而是拆成 Express Router / service 模块，接入 `mango-fleetflow-ai/server/index.js`。

### 3. Vue 前端不直接迁移

`mango-money/src/App.vue` 只作为功能参考，不直接放进 React 项目。前端需要用 React 重写组件。

### 4. 公式配置必须保留

表格管理中配置的公式不是存在 Vue 代码里，而是存在数据库中：

```text
cw_formula_field_config
```

迁移后必须继续读取这张表，并继续生成/同步：

```text
cw_formula_view_原表名
```

## 二、建议目录结构

建议先把 `mango-money` 作为迁移源放入 React 项目：

```text
mango-fleetflow-ai/
├── _migration/
│   └── mango-money/
├── server/
├── src/
├── skills/
├── create_file/
├── app_config.json
└── package.json
```

`_migration/mango-money` 只作为参考源，最终功能迁完后可以删除。

不要长期从 `_migration/mango-money` 里运行代码。

## 六、表格管理公式迁移说明

表格管理里配置的公式不属于 Vue 页面资产，属于数据库资产。

相关表：

```text
cw_table_view_config
cw_field_option_config
cw_formula_field_config
```

迁移后 React 表格管理页面仍然写入这些表。

公式视图命名规则必须保持：

```text
cw_formula_view_原表名
```

显示规则必须保持：

```text
原表字段有值 -> 显示原表字段
原表字段为空 -> 显示公式视图同名字段
```

保存规则必须保持：

- 前端提交用户修改。
- 后端写入真实业务表。
- 对本表依赖公式，后端兜底计算。
- 跨表公式通过数据库视图计算。

企业微信公式转换必须保持：

```text
[字段]="" -> empty({this.字段})
""=[字段] -> empty({this.字段})
```

不要再生成：

```text
eq({this.日期字段},"")
```

即使数据库里还有旧公式，后端 SQL 生成也必须能安全处理。

## 十一、不能丢的关键修复

迁移时必须带上当前项目已经修过的这些逻辑：

### 1. 公式视图显示规则

```text
原表字段有值 -> 原表值
原表字段为空 -> 公式视图值
```

### 2. 日期空字符串安全处理

公式 SQL 里日期参数必须安全归一化：

```text
'' -> NULL
0000-00-00 -> NULL
0000-00-00 00:00:00 -> NULL
```

### 3. eq(日期字段,"") 安全处理

即使数据库中存在旧公式：

```text
eq({this.hcsj},"")
```

后端也必须生成安全 SQL，不能生成：

```sql
base.hcsj = ''
```

### 4. 企业微信公式转换

转换时必须把：

```text
[字段]=""
""=[字段]
```

转为：

```text
empty({this.字段})
```

不要转为：

```text
eq({this.字段},"")
```

## 十二、迁移完成后的删除项

确认 React 项目功能完整后，可以删除：

```text
_migration/mango-money
mango-money/src/
mango-money/server.js
mango-money/build/
mango-money/dist/
mango-money/.runtime/
```

但删除前必须确认：

- React 项目中已有全部财务功能。
- 单据生图正常。
- 历史图片可访问。
- 公式视图可查询。
- 数据库配置无丢失。
- 线上服务已重启并验证。
